a태그란?

주로 다른 페이지나 사이트에 이동하기 위하여 사용하는 태그
연결 되어 있다는 의미로 "링크"라는 용어를 사용함!

웹 구성시 사용되는 "기본"이면서 "필수"인 태그!

  1. 1) div태그 (body를 대신하는 영역을 만들 수 있고, 각각의 section을 위한 영역도 구성할 수 있음!)
  2. 2) headline 태그 (html5에서는 "목차"생성에 필수 / xhtml, h1(로고), 영역의 제목(h2))
  3. 3) list(ol li , ul li , dl dt dd, 비슷한 객체의 값들을 나열할때 사용하는 필수 태그!)
  4. 4) img태그 (이미지를 호출하여 "변경"하거나 "교체"하는 작업을 할때 매우 중요한 태그!)
  5. 5) DB에 연결하는 태그 , control 페이지에 연결하는 태그, "form 태그"
  6. 6) 주로 서브페이지나 어떤 DB를 제외한 값들을 연결할때는 "a태그" 사용

a태그 작성시 필수 요소

  1. a태그에는 "href"라는 속성이 있음, href가 없으면 주소를 연결할 수 없을 뿐 아니라 클릭을 할 수 없음
  2. img 태그에 있는 alt, a 태그에 있는 title (대체 텍스트)
  3. 네이버, 다음, 구글의 경우 주소만 봐도 어떤 사이트인지 알 수 있음
    : 서브페이지에 연결 되어 있을 경우, subpage.html (무슨 서브페이지?)
    : src="background.jpg" , alt="과일 배경" 이미지 파일만 보고 알 수 없음, 대체 택스트 제공
    : href="subpage.html", title="sports" , 연결된 주소만 보고 알 수 없지만 title을 보면
    어떤 서브페이지 어떤 페이지로 연결하려는지 알 수 있음! (alt 처럼, 대체 텍스트 역할을 담당함)
  4. 중요) a태그 사용시, 내용이 없거나 href가 없으면 클릭이나 주소 연결을 할 수 없으므로
    반드시 필수로 작성하기
  5. 매우중요) title 속성은 "a태그 에서만" 사용하는 속성이 아님!
  6. 매우중요) a태그는 인라인 요소!
  7. img 태그 (src, alt), input 태그 (id, type, name), select 태그 ("value", id, name)
    a태그 (href, title)
  8. * 웹 표준, 웹 접근성에서 title , alt을 기입하지 않으면 "대체 텍스트" 가 없기 때문에 감점의 원인이 될 수 있다!
  9. : 웹 점검도구 (kwah, 웹접근성 연구소에서 표준법을 맞출때 사용한 도구!)
    : a태그, input, laber, img태그, select태그등 짝을 맞추는 태그나 필수 속성이 없다면
    감점의 원인이 크기 때문에 반드시 작성해야 함!
